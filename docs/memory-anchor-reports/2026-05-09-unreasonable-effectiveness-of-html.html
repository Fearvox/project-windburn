HTML as Agent Memory Anchor

Markdown is good for agent scratchpads. HTML is better when the output must become a durable object that a human can actually read, inspect, interact with, and hand to a future agent.

Core move Turn long text into spatial cognition: diagrams, tabs, annotations, exports.
Windburn angle A report is not just documentation; it can be an executable memory boundary.
Stack call Single-file HTML first. Next/shadcn later when reports become a product.
source compression interactive cognition future reuse

我们该偷的不是 HTML

这篇帖子的真正信号不是“以后都输出网页”。更深一层是：agent 产物正在从 可编辑文本转向可阅读、可验证、可交互、可重新喂给未来 session 的认知界面。

1. source 论文、X thread、repo diff、benchmark run、Research Vault note。
2. distill 提炼 thesis、claims、evidence、counter-questions、open loops。
3. visualize 用空间、颜色、流程、对照表降低人类阅读成本。
4. anchor 把最重要的判断变成下次能重新调用的 durable object。
5. boot 未来 agent 读取 HTML，继承边界、判断和下一步。

四种输出模式

同样是 HTML，但不同任务应该长得不一样。下面这个分类可以直接变成 Windburn 的 report generator 模板。

Read: paper / thread explainer

目标是让你一次读懂。适合论文、长帖、技术概念、incident postmortem。

  • 一屏 thesis + 三个关键判断。
  • 流程图解释机制，不堆段落。
  • 明确 source fact 和 Codex inference 的边界。

Review: PR / benchmark explainer

目标是让 reviewer 更快发现风险。适合代码改动、agent run、benchmark report。

  • diff annotations、severity colors、blast radius map。
  • 把 PASS/FLAG/BLOCK 和证据路径摆在同一屏。
  • 适合我们反 LGTM 的 workflow。

Operate: throwaway editor

目标是让你用手调一个复杂对象，然后导出给 agent。适合 ticket 排序、prompt tuning、config review。

  • 不是 SaaS，不追求复用。
  • 关键是末尾必须能 export JSON / prompt / diff。
  • 人类操作成为下一轮 agent input。

Remember: memory anchor

目标是让未来的你和未来的 agent 快速恢复判断。适合 Windburn / RV / paper reading / architecture decisions。

  • 必须有 source card、claim map、open loops、next prompt。
  • 把“我为什么信这个”与“我还不确定什么”分开。
  • HTML 是人读的锚；`.learning` 或 RV 是 agent 读的底座。

Stack 选择

现在别上重框架。先让 report format 长出审美和结构，再把可复用的部分产品化。

Option Best use Why / boundary
Single-file HTML 论文锚点、PR explainer、benchmark digest、个人阅读报告。 最快、最便携、可直接进 repo 和浏览器；缺点是 diff noisy，不适合作为长期 app code。
Next + shadcn 当我们要做 report gallery、search、multi-user sharing、RV backed generator。 组件化和主题很好，但会把“读一篇”变成“维护一个产品”。第二阶段再上。
Rust / Zig 本地索引器、HTML compiler、artifact packer、privacy scan、static export pipeline。 很适合做 durable toolchain；不适合作为第一版阅读界面本身。
