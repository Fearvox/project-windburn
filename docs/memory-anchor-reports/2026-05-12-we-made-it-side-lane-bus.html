Three Markers, Three Purposes

MarkerArtifact TypeWhen to Use
PARK_TO_PARENT:PARKING_NOTEFuzzy idea, observation, or half-formed thought — park for later review
DISTILL_TO_PARENT:DISTILLStructured analysis, recommendation, or evidence bundle — ready for perception
RETURN_TO_PARENT:RETURNCompleted task result — the artifact IS the deliverable

Safety Rules (Non-Negotiable)

  • No full side-chat transcripts are read, stored, or forwarded
  • No automatic source-truth promotion — human gate always required
  • Hook is path-scoped to the Windburn repo
  • Hook is no-op without an explicit relay marker
  • cwd scope check rejects non-Windburn sessions
  • Boundary note is always prepended to injected content
  • Hooks are observers only — idempotent, unordered

Boundary Hardening (Added Today)

The perception bus now detects and flags (but does not block) these boundary violations:

FlagPattern
source_truth_claimPayload claims source-truth or canonical status
pre_marker_chatterPayload opens with prose/narrative, not structured fields
post_artifact_chatterStructured artifact followed by additional prose
multiple_relay_markersPayload contains more than one marker string

Hard errors (BLOCK) remain: empty payload, invalid marker, cwd out of scope. Boundary flags are warnings — the artifact still injects but the receipt records the flag for human review.

The Loop We Proved

  1. Operator triggers a bounded Hermes side-lane run (CLI or /goal)
  2. Hermes reads protocol docs, produces DISTILL_TO_PARENT: artifact
  3. Output arrives at parent (pasted or hook-captured)
  4. Hook extracts only the marker+payload into a local-only relay queue
  5. Perception bus validates, detects boundary violations
  6. Valid artifacts are injected into a Codex thread via thread/inject_items
  7. Model can see the artifact with relay_id, metadata, and boundary note
  8. Model classifies: parking / perception candidate / belief candidate
  9. Source-truth promotion always requires human approval

What This Unlocks

A perception bus means Windburn can have multiple side-lane collaborators (Hermes, Codex, Claude Code, custom agents) feeding bounded, inspectable artifacts into a single parent thread without transcript leaks, credential exposure, or silent source-truth contamination.

The architecture is intentionally porous inbound, tight outbound: many agents can speak into the bus, but only explicit markers enter, and only the human can promote to truth.
