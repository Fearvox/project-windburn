# Field Note 000 Podcast Script

Title: HTML That Agents Can Remember.

This is Field Note zero from the personal AI research lab.

The working claim is simple:

Research Vault, to HTML, to browser, to receipt.

That chain is the strongest portable primitive I have found so far for evidence-aware agent work.

Not just Markdown. Not just a dashboard. Not just a knowledge-base connector. A chain.

Research Vault gives the agent a memory surface with health, taxonomy, search, full notes, and queue state. HTML turns the conclusion into something a human can inspect without replaying the whole session. The browser turns that artifact into a runtime: it can show screenshots, diagrams, controls, source cards, open loops, and a next-session prompt in the same object. The receipt is what lets the next person, or the next agent, ask: where did this claim come from, how fresh is it, and what still needs verification?

That is the point. The goal is not to make prettier AI output. The goal is to make the output harder to fake.

Markdown is still good. I love Markdown for scratchpads, checklists, diffs, and notes that need to stay compact. But once the work needs to be reviewed by a human, reopened by another agent, inspected in a browser, and used as durable memory, Markdown starts flattening too much context into a long scroll.

HTML changes the shape of the memory.

A browser page can hold a claim map. It can show source cards. It can label what is fact, what is inference, what is speculation, and what still needs a verification pass. It can include screenshots and diagrams without forcing the reader to imagine the state. It can expose a prompt companion, so the next agent knows not only what to do, but why the previous path worked.

This is where Research Vault matters.

For agent work, memory cannot just be vibes. It has to tell the agent whether the corpus is alive, whether analysis is stale, what taxonomy exists, and whether claims can be traced back to durable notes. A useful memory layer should answer the first agent question in under a minute: am I looking at evidence, or am I about to hallucinate from stale confidence?

That is why the basic Research Vault flow is so valuable:

Status tells the agent whether the vault is alive.
Taxonomy tells the agent where to look.
Search gives compact result rows.
Get gives the full note.
Queue state tells the agent whether missing knowledge is absent, or merely unanalyzed.

Now add HTML.

The agent can turn the evidence loop into a browser-readable object. A human can scan it. Another agent can reopen it. Browser automation can verify that it renders, that links work, that the screenshot matches the claim, and that the artifact does not leak private surface.

That last part matters a lot.

Beautiful AI output without provenance is just polish without proof. It can look finished while hiding uncertainty. The better artifact is not the smoothest one. The better artifact is the one that shows its work, shows its freshness, and makes the next verification step obvious.

So the recipe is:

First, collect the source: a repo diff, a paper, a thread, a benchmark run, or a Research Vault cluster.

Second, distill the claims: separate what the source says from what we infer.

Third, visualize the structure: timelines, matrices, diagrams, cards, receipts.

Fourth, anchor the memory: ship a single-file HTML object when portability matters.

Fifth, boot the next session: include the prompt companion, the open loops, and the evidence boundaries.

This is not a replacement for source truth. The HTML is not the truth. It is the memory anchor.

The source remains the repo, the vault, the paper, the run, the transcript, the public page. The HTML is the interface that helps humans and agents keep the source relationship visible.

That is the distinction I care about:

Do not trust the page because it is polished.
Trust the page only when it preserves provenance, freshness, replayability, and unresolved uncertainty.

In practice, this is already changing how I want agents to work.

I want less "looks good to me." I want more: here is the claim, here is the evidence, here is the stale edge, here is the next check, here is the receipt.

Anti-LGTM is not just a review rule. It is a product affordance.

The object should make lazy agreement harder.

That is why the live artifact matters. It is not just a webpage. It is a small piece of agent-native infrastructure:

Research Vault for evidence.
HTML for durable cognition.
Browser for inspection and automation.
Receipt for accountability.

That is the frame.

Research Vault, to HTML, to browser, to receipt.

More objects that future agents can actually inherit.
