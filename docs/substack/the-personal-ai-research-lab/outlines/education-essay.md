# Outline: What AI Education Needs Next Is Not More Prompts, But Better Evidence

Audience: universities and learning institutions
Audio: later, after text lock
Role: institutional bridge

## Thesis

OpenAI is asking what useful AI education looks like. From inside a real ChatGPT Edu / university workflow, the answer is not more prompt literacy alone. The next layer is evidence literacy: the habit of reading traces, claims, tool state, uncertainty, and safe next actions responsibly.

This is an essay argument, not a universal institutional proof. The local data can show aggregate usage density and review activity; it cannot prove that every university needs the same operating model.

## Structure

1. Open with the current education conversation: students as builders, agency, and rollout data.
2. State the problem: institutions can provision AI access without learning how to read what happens next.
3. Show verified aggregate usage density: active users, sessions, user messages, and review events.
4. Explain the caveat: operator-reported thread/turn figures are not verified by the public aggregate and should not be used as evidence claims.
5. Introduce evidence literacy: learning state, tool state, source/provenance, uncertainty, redaction, and safe next action.
6. Explain why rankings should be by days and workflows, never people.
7. Recommend institutional practices: public-safe dashboards, office-hour feedback loops, provenance-aware review, student agency, and redaction discipline.
8. Close by returning to agency: students should build with AI, but institutions should give them evidence surfaces rather than prompt theater.
