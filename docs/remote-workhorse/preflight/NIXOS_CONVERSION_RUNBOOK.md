the pinned `nixos-infect` script during first boot.

## Apply Gate

The staging command is:

```sh
scripts/nixos-conversion.sh \
  --apply \
  --confirm-destructive-nixos-conversion \
  --confirm-snapshot-id 227115138
```

This runs `nixos-infect` with `NO_REBOOT=1`, so the install is staged and logs
are captured before the reboot that triggers lustration.

The one-shot stage plus reboot command is:

```sh
scripts/nixos-conversion.sh \
  --apply \
  --confirm-destructive-nixos-conversion \
  --confirm-snapshot-id 227115138 \
  --reboot-after-stage \
  --confirm-lustrate-reboot
```

## Recovery

If the host does not return after reboot:

1. Use DigitalOcean console or action logs to inspect boot state.
2. Restore Droplet `568689911` from snapshot `227115138`, or create a new
   Droplet from that snapshot.
3. Remove stale local `known_hosts` only after capturing the replacement host
   key with `ssh-keyscan`.

## Stop Conditions

Return `BLOCK` before apply if:

- Snapshot `227115138` is missing.
- Strict SSH proof fails.
- The downloaded `nixos-infect` SHA-256 does not match the pinned value.
- The operator has not confirmed destructive conversion.
- There is no clear path to observe the reboot and verify NixOS identity.
